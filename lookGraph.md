# Claude 代码地图助手 - 产品设计文档
## 文档信息
- 文档名称：Claude 代码地图助手产品设计
- 技术选型：Neo4j（图数据库）+ ChromaDB（向量数据库）
- 目标场景：辅助 Claude 理解大型代码项目，替代全量代码读取，提升业务理解与任务执行效率
- 版本：V1.0

---

## 1. 产品概述
### 1.1 产品定位
Claude 代码地图助手是面向 Claude 大模型的代码项目智能辅助工具。通过**代码结构图谱**+**业务语义向量**双引擎，构建项目可视化“代码地图”，让 Claude 无需加载全量源码，即可快速掌握代码架构、依赖关系、业务含义，降低上下文开销，提升代码分析、开发、重构、问题排查的准确率与效率。

### 1.2 核心价值
1. **精简上下文**：不再向 Claude 传输整份项目代码，仅按需返回代码切片、结构关系、语义信息，规避上下文超限问题。
2. **结构化认知**：依托图数据库还原类、方法、模块间继承、调用、依赖关系，形成代码骨架地图。
3. **语义化理解**：基于向量数据库解析注释、业务描述，让大模型读懂代码业务场景，而非仅识别语法。
4. **动态同步**：支持代码增量更新，保证图谱、向量数据与本地代码实时一致。
5. **降低使用成本**：轻量组件架构，ChromaDB + Neo4j 社区版即可部署，易集成、易维护。

### 1.3 整体技术栈
| 模块 | 技术选型 | 用途 |
| ---- | ---- | ---- |
| 图数据库 | Neo4j 5.x+ | 存储代码实体、调用/继承/依赖等网状关系，构建代码结构地图 |
| 向量数据库 | ChromaDB | 存储代码注释、业务描述向量，提供自然语言语义检索 |
| 代码解析 | AST 语法解析 | 解析多语言源码，提取类、方法、注解、代码关系 |
| 服务层 | Python | 提供统一 API 服务，对接解析、双数据库、Claude 交互 |
| 交互层 | HTTP API / Claude 插件 | 对外提供查询、检索、上下文组装能力 |

## 2. 需求说明
### 2.1 基础功能需求
1. 项目代码全量解析，自动构建 Neo4j 代码关系图谱。
2. 提取类、方法注释与业务说明，向量化后存入 ChromaDB。
3. 支持手动/自动触发代码增量扫描，更新图谱与向量数据。
4. 提供代码结构查询、业务语义检索、代码上下文切片三大核心能力。
5. 生成项目极简摘要，辅助 Claude 建立全局架构认知。

### 2.2 进阶功能需求
1. 支持业务标签管理，按业务域（订单/支付/用户等）归类代码实体。
2. 代码变更影响范围分析，判断修改波及的类、方法、模块。
3. 调用链路全链路查询与梳理。
4. 多项目隔离管理，支持切换不同代码工程。
5. 链路可视化数据输出。

### 2.3 非功能需求
1. **性能**：增量解析秒级完成，单条查询响应 < 500ms。
2. **兼容性**：优先支持 Java、Python、JavaScript 主流开发语言。
3. **稳定性**：数据库数据持久化，异常场景数据不丢失。
4. **易用性**：标准化 HTTP API，低门槛对接 Claude。

## 3. 系统整体架构
### 3.1 四层分层架构
```
┌─────────────────────────────────────────────────────┐
│ 应用交互层 │
│ Claude 插件 / 第三方调用 API │
└───────────────────────┬─────────────────────────────┘
│
┌───────────────────────▼─────────────────────────────┐
│ 核心服务层 │
│ 结构查询服务 | 语义检索服务 | 上下文组装服务 │
│ 项目摘要服务 | 影响分析服务 | 任务调度服务 │
└───────────────────────┬─────────────────────────────┘
│
┌───────────────────────▼─────────────────────────────┐
│ 数据存储层 │
│ Neo4j（图数据库） ChromaDB（向量数据库） │
└───────────────────────┬─────────────────────────────┘
│
┌───────────────────────▼─────────────────────────────┐
│ 代码解析层 │
│ AST 解析 | 实体提取 | 关系提取 | 注释提取 | 标签提取 │
└─────────────────────────────────────────────────────┘
```

### 3.2 各层级职责
1. **代码解析层**
   扫描本地代码文件，基于 AST 解析语法树，提取项目、模块、类、方法等实体，以及继承、实现、调用、依赖等关系；同时提取注释、文档、业务描述。

2. **数据存储层**
   - Neo4j：持久化代码实体与实体间关系，作为**代码结构地图**。
   - ChromaDB：对注释、业务文本做向量化存储，提供**语义检索能力**。

3. **核心服务层**
   封装所有业务逻辑，对外提供统一接口，包含查询、检索、上下文拼接、摘要生成、增量同步等能力。

4. **应用交互层**
   对外暴露 HTTP API，可直接对接 Claude 插件、客户端、脚本等调用方。

## 4. 数据模型设计
### 4.1 Neo4j 图数据模型（节点 + 关系）
#### 4.1.1 节点（Node）定义
| 节点类型 | 标识 | 核心属性 |
| ---- | ---- | ---- |
| 项目 | Project | project_id、name、path、tech_stack、create_time |
| 模块 | Module | module_id、name、business_tag、project_id |
| 类/接口 | Class | class_id、name、file_path、comment、module_id、type(类/接口) |
| 方法 | Method | method_id、name、params、return_type、comment、class_id |

#### 4.1.2 关系（Relationship）定义
| 关系类型 | 关联两端 | 关系说明 |
| ---- | ---- | ---- |
| BELONGS_TO | Class → Module <br> Method → Class | 归属关系：类属于模块、方法属于类 |
| EXTENDS | Class → Class | 类继承关系 |
| IMPLEMENTS | Class → Class(接口) | 类实现接口 |
| CALLS | Method → Method | 方法调用关系 |
| DEPENDS_ON | Class → Class <br> Module → Module | 代码/模块依赖关系 |

### 4.2 ChromaDB 向量数据模型
ChromaDB 按集合（Collection）隔离数据，统一存储向量、原文、元数据。

#### 4.2.1 集合 1：`code_semantics`（代码语义主集合）
- 存储内容：类注释、方法注释、业务描述、功能说明
- 向量来源：注释 + 业务文本 Embedding 向量化
- 元数据字段：
  - entity_id：关联 Neo4j 实体ID
  - entity_type：类型（class / method）
  - file_path：文件路径
  - module_name：所属模块
  - business_tag：业务标签

#### 4.2.2 集合 2：`project_overview`（项目摘要集合）
- 存储内容：项目架构说明、核心模块介绍、主业务流程
- 向量来源：项目摘要文本 Embedding 向量化
- 元数据字段：project_id、update_time

## 5. 核心功能详细设计
### 5.1 代码地图构建与更新（Neo4j 核心）
#### 5.1.1 全量初始化
1. 选定本地代码项目路径，触发全量扫描。
2. AST 遍历所有文件，提取 Project / Module / Class / Method 实体。
3. 解析继承、实现、调用、依赖关系，生成对应关系边。
4. 批量写入 Neo4j，完成初代代码地图构建。

#### 5.1.2 增量更新
1. 监听文件变更（新增/修改/删除）。
2. 仅解析变动文件，局部更新 Neo4j 节点与关系。
3. 同步删除失效节点与关系，保证图谱准确性。

### 5.2 语义索引构建与检索（ChromaDB 核心）
1. 自动抓取代码中单行/多行注释、文档字符串。
2. 文本向量化后存入 ChromaDB，并绑定对应实体元数据。
3. 支持**自然语言检索**：输入业务描述，召回相似度最高的代码实体。
4. 变更文件同步更新对应向量数据。

### 5.3 结构查询服务
面向 Claude 提供代码结构查询能力：
- 查询指定类的继承、实现、依赖关系
- 查询指定方法的完整调用链路
- 查询某个模块下所有类与核心方法
- 分析某个类/方法修改后的影响范围

### 5.4 语义检索服务
- 根据业务描述、功能需求，模糊匹配对应代码位置
- 支持按模块、业务标签做检索过滤
- 召回结果关联文件路径、注释、所属模块

### 5.5 最小上下文组装（核心优化）
拒绝返回完整源码文件，**按需拼接最小有效上下文**：
1. 目标方法/类源码
2. 直接上游调用方、下游被调用方（1~2层）
3. 关键注释与业务说明
4. 所属模块与业务标签

### 5.6 项目摘要服务
自动生成精简项目介绍，每次对话优先推送给 Claude：
- 项目技术栈
- 整体架构简述
- 核心业务模块清单
- 核心业务主流程

## 6. 核心业务流程
### 6.1 项目初始化流程
1. 用户配置本地代码项目路径，执行「初始化地图」。
2. 代码解析层全量扫描代码，提取实体与关系。
3. 实体+关系写入 Neo4j，构建代码结构图谱。
4. 提取注释与业务文本，向量化后写入 ChromaDB。
5. 自动生成项目摘要，存入向量库。
6. 初始化完成，对外提供查询服务。

### 6.2 代码变更同步流程
1. 监控到本地文件新增/修改/删除。
2. 增量解析变动代码。
3. 更新 Neo4j 对应节点、关系。
4. 更新/删除 ChromaDB 中对应向量数据。
5. 按需刷新项目摘要。

### 6.3 Claude 调用全流程
1. 用户向 Claude 发起代码相关提问/任务。
2. Claude 调用本工具 API。
3. 工具判断请求类型：
   - 查结构 → 查询 Neo4j
   - 查业务功能 → 检索 ChromaDB
4. 组装**最小代码上下文+结构信息+语义说明**。
5. 将结果返回给 Claude。
6. Claude 基于精简上下文执行任务并回复用户。

## 7. 接口设计（HTTP API 精简版）
### 7.1 项目管理接口
* 初始化项目地图<br>POST /api/project/init
* 触发增量更新<br>POST /api/project/update
* 获取项目摘要<br>GET /api/project/summary


### 7.2 结构查询接口（Neo4j）
* 查询类的关联关系<br>GET /api/structure/class/{classId}/relation
* 查询方法调用链路<br>GET /api/structure/method/{methodId}/callchain
* 查询模块下所有类<br>GET /api/structure/module/{moduleId}/classes
* 查询代码修改影响范围<br>GET /api/structure/impact/{entityType}/{entityId}


### 7.3 语义检索接口（ChromaDB）
* 语义检索
```curl
POST /api/semantic/search
Body: {
"query": "支付失败重试逻辑",
"module": "支付模块",
"top_k": 5
}
```



### 7.4 上下文获取接口
* 获取方法精简上下文<br>GET /api/context/method/{methodId}
* 获取类精简上下文<br>GET /api/context/class/{classId}

## 8. 部署环境要求
### 8.1 基础环境
- 运行语言：Python 3.9 ~ 3.12
- 内存：最低 4GB，推荐 8GB+（大型项目）
- 操作系统：Windows / Linux / macOS

### 8.2 中间件部署
1. **Neo4j**
   - 版本：5.x 社区版
   - 开启持久化存储，配置账号密码
2. **ChromaDB**
   - 最新稳定版，本地文件存储模式（开箱即用，无需额外服务）

## 10. 产品优势总结
1. **轻量化部署**：ChromaDB 本地运行、Neo4j 社区版免费可用，部署简单。
2. **双引擎互补**：图数据库管结构、向量库管语义，完整还原代码全貌。
3. **极致省上下文**：只返回必要代码片段，彻底解决大模型超长代码加载问题。
4. **动态可维护**：代码变更自动同步，地图永远保持最新。
5. **高扩展性**：分层架构设计，后续新增语言、功能、可视化能力均可无缝扩展。
